Git e GitHub

Explicando um pouco sobre o Git e GitHub.

Introdução:

Seja bem vindo caro leitor! Este artigo que você irá ler será um dos mais importantes para a sua base de conhecimento sobre o mundo dos desenvolvedores de uma maneira generalizada, pois iremos lhe falar sobre uma das ferramentas mais importantes do mundo dev no século XXI: o Git.

O Git é uma ferramenta de versionamento. Sua principal função é auxiliar o trabalho dos programadores com sua capacidade de retornar a um ponto específico(versão) do desenvolvimento de algum software ou o que quer que você esteja produzindo. Um número impressionante de projetos de software depende do Git para controle de versão, incluindo projetos comerciais e de código-fonte aberto. É muito vantajoso utilizar o Git quando se está trabalhando em equipe pois facilita, promove a cooperação entre os integrantes e é mais recomendado ainda utilizar o GitHub, como iremos lhe informar mais a frente.

Um pouco da história:

O Git é um projeto de código aberto maduro e com manutenção ativa desde 2005. Seu criador foi o próprio Linus Towarlds! Sim, o mesmo desenvolvedor do kernel Linux! Tudo começou quando houve um rompimento da relação entre a comunidade que desenvolvia o Kernel do Linux e a BitKeeper(é um sistema de controle de versão e é um software proprietário). A revogação de um insento de pagamento e o rompimento da relação com a BitKeeper, levou Linus Torvalds a desenvolver seu próprio software de versionamento, baseado nas experiências de uso da BetKeeper, e visando obter uma melhor performance dentre todos os sistemas de controle de versão da época. Então em 2005 surge o Git! Git de acordo com o próprio Linus, é uma gíria britânica que significa “cabeça dura”.

Características

O Git foi projetado para ter flexibilidade, segurança e principalmente: Desempenho. Dentre as qualidades do mesmo, se destacam a capacidade de lidar com projetos de grande porte de maneira eficiente e possuir suporte robusto a desenvolvimento não linear.

GitHub

Se você é um iniciante no mundo da progamação, é provável que você já tenha ouvido falar desse termo. Se não, aqui vai a explicação:

GitHub é uma plataforma de hospedagem de código-fonte e arquivos com controle de versão usando o Git. Ele permite que programadores, utilitários ou qualquer usuário cadastrado na plataforma contribuam em projetos privados e/ou Open Source de qualquer lugar do mundo.

GitHub, como já diz na sua logo, “Where the world builds software”, traduzindo para o nosso idioma, “Onde o mundo constrói software”, é o local onde mais de 80 milhões de desenvolvedores desenvolvem softwares, juntos.

O GitHub hospeda mais de 100 milhões (https://venturebeat.com/2018/11/08/github-passes-100-million-repositories/) de repositórios, com a maior parte deles sendo projetos de código aberto. Essa estatística mostra que o GitHub está entre os clientes de Git GUI (https://www.hostinger.com.br/tutoriais/git-gui) mais populares, e também porque é usado por vários profissionais e grandes empresas https://stackshare.io/github ,como a Hostinger.

Se você deseja aprender a trabalhar com o Git e o GitHub, lhe recomendamos o curso do professor Gustavo Guanabara, da empresa Curso em Vídeo: https://www.cursoemvideo.com/curso/curso-de-git-e-github/.
